# Frontend Audit — 2026-03-03

## Что проверил
- Общий layout и контейнеризацию страниц (`.app`, `.plate-page`, `.admin`, `.users`).
- Единообразие и поведение header/dashboard.
- Всплывающее меню по кнопке «⋮» (визуал, UX, доступность, поведение).
- Консистентность интеракций hover/focus/active.

## Основные проблемы (до правок)
1. **Header menu выглядел утилитарно и «плоско»**: маленькая глубина, мало иерархии, слабые состояния `focus-visible`.
2. **Меню открывалось/закрывалось только по клику**, без управления клавишей `Esc` и без `aria-expanded` на триггере.
3. **Визуальный шум в header**: рядом с меню была отдельная ссылка «Выйти», которая дублирует пункт в dropdown.
4. **Недостаточно «дорогой» UI-подачи**: не хватало аккуратной типографики и micro-interactions у dropdown-меню.

## Что улучшено
1. **Прокачан вид меню «⋮»**:
   - более аккуратная кнопка-триггер с hover/focus/active состояниями;
   - мягкая тень и градиент у dropdown;
   - улучшенные отступы, скругления, читаемость секций.
2. **Улучшено UX/доступность**:
   - `aria-expanded` синхронизируется с состоянием меню;
   - закрытие меню по `Esc`;
   - сохранена логика закрытия при клике вне меню.
3. **Снижен шум в шапке**:
   - ссылка «Выйти» в header скрыта (основной путь — через dropdown).
4. **Усилен «premium» стиль**:
   - добавлены тонкие переходы, более качественная иерархия в меню и акцентирование активного пункта.

## Дальнейшие рекомендации (next step)
- Перейти на единый компонентный подход (например, шаблон header + partial), чтобы не дублировать HTML-шапку в множестве страниц.
- Вынести дизайн-токены в отдельный файл и разделить `styles.css` на модули (layout/header/forms/tables/pages).
- Добавить визуальные regression-check скриншоты по ключевым страницам в CI.
