# Рефакторинг фронтенда (2026)

Краткое описание изменений без нарушения бизнес-логики и API.

## Что сделано

### 1. Дизайн-система (styles.css)

- **Токены**: единая палитра (primary, neutrals, danger), шкала отступов 4–40px (8px сетка), радиусы 8/12/16px, мягкие тени (без «2018»).
- **Типографика**: H1, H2, section, body, caption; межстрочный интервал 1.5; контраст текста сохранён для читаемости.
- **Цвета**: один акцент (зелёный), нейтральные серые, без градиентов и лишних бордеров.

### 2. Layout

- Все страницы с контентом используют общий максимум ширины 1280px и padding 24px.
- Шапка: единый блок с навигацией, скругление 16px, лёгкая тень и бордер.
- Чёткое разделение: навигация в шапке → рабочая зона (main) → футер (где есть).

### 3. Компоненты

- **Кнопки**: primary / secondary / danger, единые размеры и hover/focus (в т.ч. focus-visible для клавиатуры).
- **Карточки**: единый .card с отступами 24px, скругление 16px, лёгкий hover.
- **Формы**: сохранены все id и классы для app.js (form-section, form-section__body--closed, field__input и т.д.), улучшены отступы и focus.
- **Таблицы**: общие стили для .plate-table, .plate-cash-table, .cash-table, .admin table, .users .users-table.
- **Модалки**: один паттерн .modal + .modal__box; отображение по-прежнему через JS (display или classList.add('show')).

### 4. Удалён визуальный шум

- Убраны инлайновые `<style>` со всех страниц; стили перенесены в styles.css.
- Единые отступы между секциями, меньше «заборов» из бордеров.
- Вторичные действия (ссылки «Выйти», отмена в модалках) визуально приглушены, но остаются доступными.

### 5. UX

- **Hover**: у ссылок в навбаре, кнопок, строк истории, строк таблиц.
- **Focus**: outline снят, вместо него box-shadow (focus-visible) для клавиатурной навигации.
- **Transition**: 150–200 ms на кнопках, ссылках, карточках, полях ввода.
- **Читаемость**: воздух 16–24px, группировка связанных блоков (форма заказа → итого → действия).

### 6. Файловая структура

- **Без изменений**: index.html, plate-operator.html, plate-cash.html, cash-shifts.html, warehouse.html, admin.html, users.html, account.html, login.html, app.js, auth.js.
- **Один общий стиль**: styles.css (все страницы подключают только его + шрифты).
- Все id и классы, которые использует JS (getElementById, querySelectorAll, classList), сохранены.

## Что не менялось

- API и эндпоинты.
- Логика в app.js и auth.js (в т.ч. создание заказа, оплата, история, печать).
- Логика в инлайновых скриптах (plate-operator, plate-cash, cash-shifts, account, admin, users, login).
- Набор страниц и навигационные ссылки.
- Поведение модалок (открытие/закрытие через существующий JS).

## Как проверить

1. Открыть index.html (форма заказа): заполнить клиента, добавить документы, «Принять наличные», «Печать документов», история заполнения по клику.
2. Вход (login.html), меню (account.html), смена пароля в модалке.
3. Павильон 2: список заказов, «Выдано», «Доплата» (модалка), заявление на номера.
4. Касса номеров и Касса: добавление строк, редактирование, итого.
5. Склад: остаток, пополнение, брак.
6. Админка: аналитика, заказы, сотрудники, прейскурант, модалка деталей заказа.
7. Аккаунты: список, добавление/редактирование в модалке, деактивация/восстановление.

Интерфейс рассчитан на комфортную работу в течение смены: меньше устаёт глаз, понятная иерархия и предсказуемые реакции на действия.
